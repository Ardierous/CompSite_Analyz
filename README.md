# Анализ корпоративных сайтов

Веб-приложение для комплексного анализа корпоративных сайтов с использованием AI-агентов на базе CrewAI.

## Описание

Приложение автоматически анализирует корпоративные сайты и генерирует подробные отчеты, включающие:

- Основную информацию о компании
- Продукты и услуги
- Организационную структуру
- Финансовые показатели
- Стратегические направления
- Конкурентный анализ
- Риски и возможности
- Инвестиционные перспективы

## Возможности

- 🚀 **Минималистичный веб-интерфейс** - простой и современный дизайн
- 🤖 **AI-анализ** - автоматический сбор и анализ информации с помощью трех специализированных агентов
- 📊 **Детальные отчеты** - структурированные отчеты на русском языке
- ⚡ **Асинхронная обработка** - анализ выполняется в фоновом режиме с отслеживанием прогресса
- 📈 **Прогресс в реальном времени** - плавная визуализация процесса анализа с равномерным движением прогресс-бара
- 💰 **Отслеживание расходов** - автоматический расчет стоимости анализа через ProxyAPI в рублях
- 📄 **Экспорт в DOCX** - возможность экспортировать результаты анализа в формат Word
- 🔄 **Множественные итерации** - готовность к анализу нескольких сайтов подряд без перезапуска
- 📝 **Логирование** - подробное логирование всех процессов в файлы
- 🛡️ **Умная обработка ошибок** - корректная обработка недоступных сайтов и ошибок API
- 🔒 **Защита от повторного запуска** - автоматическое завершение предыдущих экземпляров приложения
- 🎨 **Цветной вывод в терминале** - визуальная индикация статуса запуска и завершения
- 🔌 **Гибкая настройка портов** - поддержка запуска на любом свободном порту
- 🌐 **Playwright (опционально)** - скрапинг через Chromium для сайтов с защитой от ботов; при установке не используется при конвертации MD → DOCX
- 📝 **MD → DOCX** - Pandoc или собственный алгоритм (python-docx); для Pandoc нужен отдельно установленный [Pandoc](https://pandoc.org/installing.html)

## Технологии

- **Backend**: Python 3.11, Flask 3.1+
- **AI Framework**: CrewAI 0.80.0+
- **Frontend**: HTML5, CSS3, JavaScript (Vanilla)
- **LLM**: OpenAI GPT-4 через ProxyAPI
- **API Proxy**: ProxyAPI (proxyapi.ru) для доступа к OpenAI API
- **Скрапинг**: crewai[tools] (HTTP), при необходимости **Playwright** + Chromium для сайтов с антиботом
- **Экспорт**: python-docx; опционально Pandoc для маршрута «MD → DOCX (Pandoc)»
- **Containerization**: Docker (в образе: Pandoc, Playwright + Chromium, переменная `PLAYWRIGHT_BROWSERS_PATH`)
- **NumPy**: версия <2.0 (совместимость с различными архитектурами)

## Установка

### Требования

- **Python 3.11** (обязательно, проект настроен только на эту версию)
- pip (менеджер пакетов Python)
- OpenAI API ключ (или совместимый сервис)
- Docker (опционально, для контейнеризации)

**Важно:** Проект использует только Python 3.11. Использование других версий может привести к ошибкам при установке зависимостей.

### Шаги установки

1. **Клонируйте репозиторий или скачайте проект**

2. **Убедитесь, что используется Python 3.11:**

   ```bash
   python --version  # Должно быть Python 3.11.x
   ```

   Если используется другая версия, используйте:

   ```bash
   py -3.11 --version  # Windows
   python3.11 --version  # Linux/Mac
   ```

3. **Установите зависимости:**

   ```bash
   pip install -r requirements.txt
   ```

   **Важно:** `requirements.txt` включает `crewai[tools]>=0.80.0` с инструментами для веб-скрапинга. Если установка завершилась с предупреждением, убедитесь, что все зависимости установлены корректно.

4. **Playwright и браузер (рекомендуется для сайтов с защитой)**

   Пакет `playwright` ставится из `requirements.txt`, но бинарник Chromium нужно скачать отдельно. Без этого браузерный скрапинг отключён: обычный HTTP-скрапинг часто получает только страницу с cookie или блокировку (например, у части корпоративных сайтов).

   ```bash
   playwright install chromium
   ```

   На **Linux / VPS** при ошибках о недостающих библиотеках:

   ```bash
   playwright install-deps
   ```

   Playwright **вызывается только при анализе сайта** (задача Crew AI со скрапингом), а **не** при конвертации MD → DOCX.

5. **Настройте переменные окружения:**

   Создайте файл `.env` в корне проекта (шаблон: `env.example` или `.env.example` для HTTPS-стека в корне):

   ```bash
   cp env.example .env
   # или: cp .env.example .env
   ```

   Затем отредактируйте `.env` и укажите свои значения:

   ```env
   # Ключ ProxyAPI (можно использовать OPENAI_API_KEY или PROXYAPI_KEY)
   OPENAI_API_KEY=your_proxyapi_key_here
   OPENAI_API_BASE=https://api.proxyapi.ru/openai/v1

   # Опционально: настройки Flask
   FLASK_HOST=127.0.0.1  # или 0.0.0.0 для доступа извне
   FLASK_PORT=5000       # можно указать любой свободный порт (например, 5001, 8000, 8080)
   FLASK_DEBUG=True

   # Пароль для разблокировки анализа сайта (единый источник — только .env)
   APP_ACCESS_PASSWORD=change_me
   # Код входа в режим смены пароля (необязательно; по умолчанию 123654+)
   # ADMIN_KEY=123654+
   ```

   Важно: пароль входа читается только из `APP_ACCESS_PASSWORD`. Хранение/смена пароля через файл и API отключены, чтобы не было расхождений между локальной средой и VPS.

   **Примечание о портах:**

   - По умолчанию используется порт **5000**
   - Можно указать любой свободный порт через переменную `FLASK_PORT`
   - Полезно, если нужно запустить несколько экземпляров на одной машине или порт 5000 занят
   - Примеры: `FLASK_PORT=5001`, `FLASK_PORT=8000`, `FLASK_PORT=8080`

   **Важно для отслеживания расходов:**

   - В личном кабинете ProxyAPI перейдите в раздел "Ключи API"
   - Выберите ваш ключ и нажмите "Редактировать"
   - Включите разрешение "Запрос баланса"
   - Без этого разрешения стоимость анализа не будет отображаться

   **Безопасность:**

   - Файл `.env` автоматически исключен из Git (указан в `.gitignore`)
   - Никогда не коммитьте файл `.env` в репозиторий
   - Ключи API передаются только через переменные окружения
   - В Docker образе `.env` не копируется (исключен через `.dockerignore`)

### Конвертация MD → DOCX

- **Pandoc** — в интерфейсе «MD → DOCX (Pandoc)»; на машине должен быть установлен исполняемый файл `pandoc` (в Docker-образе Pandoc уже есть).
- **Свой алгоритм** — «MD → DOCX (свой алгоритм)»: разбор Markdown в python-docx. Длинное тире в тексте при выводе нормализуется до обычного тире (— → –).

## Запуск

### Локальный запуск

1. **Запустите приложение:**

   ```bash
   python main.py
   ```

   Или с явным указанием версии:

   ```bash
   py -3.11 main.py  # Windows
   python3.11 main.py  # Linux/Mac
   ```

   **Особенности запуска:**
   - При запуске приложение автоматически проверяет наличие предыдущих экземпляров
   - Если обнаружен запущенный процесс - он принудительно завершается
   - Проверяется занятость порта (до 10 попыток освобождения)
   - Создается PID файл (`.app.pid`) для отслеживания запущенного экземпляра
   - В терминале отображаются цветные сообщения:
     - 🟢 **Зеленым** - сообщения о запуске приложения
     - 🔴 **Красным** - сообщения о завершении процессов
     - 🟡 **Желтым** - предупреждения

2. **Откройте браузер** и перейдите по адресу:

   ```text
   http://localhost:5000
   ```

   Или по указанному порту, если изменили `FLASK_PORT` в `.env`

## AI-Агенты

Приложение использует три специализированных AI-агента:

1. **Web Scraper Agent** - собирает информацию ТОЛЬКО с указанного корпоративного сайта (инструменты: HTTP-скрапинг; при установленном Playwright — Chromium для обхода части антибот-защиты и JS-контента)
2. **Data Analyzer Agent** - анализирует собранные данные и выявляет паттерны (работает только с данными с указанного сайта)
3. **BI Engineer Agent** - формирует итоговый структурированный отчет (на основе данных только с указанного сайта)

**Важно:** Все агенты работают исключительно с информацией, полученной с указанного сайта. Использование любых других источников (поиск в интернете, базы данных, внешние сайты) запрещено.

Если отчёт пустой или содержит только уведомление о cookie, чаще всего сайт отдаёт ботам «пустую» страницу: выполните `playwright install chromium` (и при необходимости `playwright install-deps` на Linux) и повторите анализ.

## Разработка

### Запуск в режиме отладки

Приложение запускается с включенным режимом отладки по умолчанию. Для продакшена измените в `main.py`:

```python
app.run(debug=False, host='0.0.0.0', port=5000)
```

### Структура API

- `GET /` - главная страница
- `POST /api/analyze` - запуск анализа сайта
  - Body: `{"url": "https://example.com"}`
  - Response: `{"task_id": "...", "status": "processing", "message": "Анализ запущен"}`
- `GET /api/status/<task_id>` - получение статуса анализа
  - Response: `{"status": "processing|completed|error", "progress": 0-100, "message": "...", "result": {...}, "cost": 0.1234}`
- `GET /api/export/<task_id>` - экспорт результатов в DOCX
  - Response: файл Word для скачивания

### Логирование

Все процессы логируются в файлы в директории `logs/`:

- Формат файла: `app_YYYYMMDD.log`
- Уровни логирования: INFO, WARNING, ERROR
- Логи включают:
  - Запуск и остановку приложения
  - Запуск и завершение анализов
  - Этапы выполнения анализа
  - Ошибки и предупреждения
  - Информацию о стоимости анализа

### Отслеживание расходов

Приложение автоматически отслеживает расходы на использование OpenAI API через ProxyAPI:

- Получает начальный баланс перед анализом
- Вычисляет стоимость после завершения анализа
- Отображает стоимость в рублях с точностью до 2 знаков после запятой
- Требуется разрешение "Запрос баланса" в настройках API ключа ProxyAPI

**Эндпоинт для баланса:** `https://api.proxyapi.ru/proxyapi/balance`

### Обработка ошибок

Приложение корректно обрабатывает различные ситуации:

- **403 Forbidden**: Предупреждение, но анализ продолжается (CrewAI может обойти защиту)
- **404 Not Found**: Блокирует анализ (сайт не существует)
- **5xx Server Error**: Блокирует анализ (временная недоступность)
- **Таймауты**: Информативное сообщение об ошибке
- **Ошибки подключения**: Детальные сообщения для диагностики

## Защита от повторного запуска

Приложение имеет встроенную защиту от одновременного запуска нескольких экземпляров:

1. **Проверка PID файла** - при запуске проверяется наличие файла `.app.pid`

   - Если файл существует и процесс запущен - предыдущий экземпляр завершается
   - Если процесс не запущен - старый PID файл удаляется

2. **Проверка порта** - дополнительная проверка занятости порта

   - Если порт занят - все процессы на порту завершаются
   - До 10 попыток освобождения порта с паузами между попытками

3. **Автоматическое создание PID файла** - при успешном запуске создается `.app.pid`

   - Содержит PID текущего процесса
   - Удаляется при корректном завершении приложения

**Важно:**

- PID файл создается автоматически и не должен редактироваться вручную
- При аварийном завершении PID файл может остаться - он будет автоматически очищен при следующем запуске
- Для принудительного завершения используйте скрипт `scripts/stop_app.py`

## Устранение проблем

### Приложение не запускается, порт занят

Если при запуске появляется сообщение о том, что порт занят:

1. **Используйте скрипт для завершения:**

   ```bash
   python scripts/stop_app.py
   ```

2. **Или измените порт в `.env`:**

   ```env
   FLASK_PORT=5001  # или другой свободный порт
   ```

3. **Или завершите процесс вручную:**

   ```bash
   # Windows
   netstat -ano | findstr :5000
   taskkill /PID <PID> /F
   
   # Linux/Mac
   lsof -ti:5000 | xargs kill -9
   ```

### Контейнер не доступен извне

Если контейнер запущен, но недоступен по сети:

1. Проверьте, что порт проброшен: `docker ps` должен показывать `0.0.0.0:5000->5000/tcp`
2. Убедитесь, что `FLASK_HOST=0.0.0.0` в переменных окружения
3. Проверьте настройки файрвола на сервере

### CrewAI не доступен

Если при запуске появляется ошибка "CrewAI не доступен":

1. **Проверьте установку CrewAI:**

   ```bash
   pip install "crewai[tools]>=0.80.0"
   ```

   **Важно:** `requirements.txt` уже содержит `crewai[tools]>=0.80.0`, но если установка не прошла, выполните команду выше.

2. **Проверьте логи** в файле `.cursor/debug.log` для детальной информации

3. **Проверьте переменные окружения:**
   - Убедитесь, что `OPENAI_API_KEY` установлен в `.env`
   - Проверьте, что `OPENAI_API_BASE` указан правильно

4. **Проверьте зависимости:**

   ```bash
   pip install -r requirements.txt
   ```

5. **Для Docker контейнеров:**
   - Убедитесь, что образ пересобран после изменений в `requirements.txt`
   - Для продакшена: пересоберите и запушьте образ в Docker Hub, затем обновите на VPS через `docker-compose pull`

### Предупреждение «Playwright-инструменты недоступны» или пустой скрапинг

- Установите браузер для Playwright: `playwright install chromium`
- На сервере Linux при сбоях запуска Chromium: `playwright install-deps`
- Сообщение при старте приложения не блокирует работу: без Playwright анализ всё равно идёт через HTTP-скрапинг, но часть сайтов может не отдавать контент

### Стоимость анализа не отображается

Если стоимость анализа не показывается:

1. Убедитесь, что в настройках API ключа ProxyAPI включено разрешение "Запрос баланса"
2. Проверьте, что ключ API имеет доступ к эндпоинту баланса
3. Проверьте логи приложения для детальной информации
